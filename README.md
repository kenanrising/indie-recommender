# Indie Music Recommender — Backend

> Deep audio embedding recommendations for independent artists.  
> Built with [LAION-CLAP](https://github.com/LAION-AI/CLAP) + FastAPI, deployable on Render in minutes.

---

## Architecture at a Glance

```
┌─────────────────────────────────────────────────────────────┐
│  OFFLINE (run once)                                         │
│                                                             │
│  tracks.csv  ──►  build_embeddings.py  ──►  embeddings.npy  │
│                        │                    track_index.csv │
│                   LAION-CLAP model                          │
│                (512-dim audio vectors)                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼  (embeddings committed/uploaded to server)
┌─────────────────────────────────────────────────────────────┐
│  ONLINE (FastAPI server)                                    │
│                                                             │
│  Adalo App  ──►  POST /recommend_deep  ──►  cosine sim      │
│                    { track_id, k }          (NumPy)         │
│                                         ──►  top-k results  │
└─────────────────────────────────────────────────────────────┘
```

## Model Choice: LAION-CLAP (music checkpoint)

| Criterion | Choice | Why |
|---|---|---|
| Model | LAION-CLAP | Contrastively pretrained on audio+text; music-specific checkpoint |
| Checkpoint | `music_audioset_epoch_15_esc_90.14.pt` | Highest GTZAN genre accuracy (71%) among available CLAP weights |
| Embedding dim | 512 | Compact enough for in-memory similarity on thousands of tracks |
| License | Apache 2.0 | No commercial restrictions |
| Alternatives considered | MERT-v1-330M | Better for genre/tag classification; overkill for similarity search |

## CSV Format

Minimum required columns:

```csv
track_id,title,artist,audio_file_path
TRK001,Midnight Bloom,Echo Valley,/data/audio/midnight_bloom.mp3
TRK002,Static Dream,Neon Wolves,/data/audio/static_dream.wav
```

Optional columns you can add later (do not break anything):

| Column | Type | Used for |
|---|---|---|
| `bpm` | float | Future hybrid scoring |
| `energy` | float 0–1 | Future hybrid scoring |
| `mood` | string | Future filter / display |
| `genre` | string | Future filter |
| `audio_url` | string | If files are remote (requires URL-fetch logic in pipeline) |

## Step-by-Step: Local Development

### 1. Clone and install

```bash
git clone https://github.com/yourname/indie-recommender.git
cd indie-recommender

python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Prepare your CSV

Edit `data/tracks_sample.csv` or point to your own file.  
`audio_file_path` must be absolute paths to real `.mp3` or `.wav` files on disk.

### 3. Build embeddings (run once)

```bash
python scripts/build_embeddings.py \
    --csv data/tracks_sample.csv \
    --out embeddings/
```

On first run this downloads the CLAP checkpoint (~600 MB) to `~/.cache/laion_clap/`.  
Subsequent runs reuse the cached model.

Output:
- `embeddings/embeddings.npy`  — the raw embedding matrix
- `embeddings/track_index.csv` — the metadata index

### 4. Start the API locally

```bash
cp .env.example .env
python api.py
```

The API starts at `http://localhost:8000`.  
Interactive docs at `http://localhost:8000/docs`.

### 5. Test it

```bash
curl -X POST http://localhost:8000/recommend_deep \
  -H "Content-Type: application/json" \
  -d '{"track_id": "TRK001", "k": 5}'
```

Expected response:
```json
{
  "results": [
    {"track_id": "TRK004", "title": "Low Signal",    "artist": "Mara Blue",     "score": 0.912},
    {"track_id": "TRK002", "title": "Static Dream",  "artist": "Neon Wolves",   "score": 0.887},
    {"track_id": "TRK003", "title": "Coastal Drive", "artist": "The Paperbacks","score": 0.841},
    {"track_id": "TRK005", "title": "Velvet Underground","artist": "Cass & The Tide","score": 0.798},
```

## Deployment: Render.com (Recommended)

### Prerequisites
- GitHub account with this repo pushed
- [Render](https://render.com) free account

### Steps

1. **Push to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/yourname/indie-recommender.git
   git push -u origin main
   ```

2. **Upload embeddings to the Render disk** (two options):

   **Option A — Commit to repo (simplest for small catalogs < 100 MB)**
   ```bash
   # Uncomment the embeddings lines in .gitignore, then:
   git add embeddings/
   git commit -m "Add prebuilt embeddings"
   git push
   ```

   **Option B — Render Persistent Disk (for large catalogs)**  
   After first deploy, SSH into the Render instance and run:
   ```bash
   # From Render shell:
   python scripts/build_embeddings.py --csv data/tracks.csv --out embeddings/
   ```
   The `render.yaml` already mounts a 1 GB disk at `embeddings/`.

3. **Connect Render to your GitHub repo**
   - Go to [render.com/dashboard](https://dashboard.render.com)
   - New → Web Service → Connect your repo
   - Render auto-detects `render.yaml` — no manual config needed

4. **Add environment variable** in Render dashboard:
   - Key: `EMBEDDINGS_DIR` / Value: `embeddings/`

5. **Deploy** — Render builds, installs deps, and starts the server.

6. **Your public URL** will be:
   ```
   https://indie-recommender-api.onrender.com
   ```
   Your `/recommend_deep` endpoint:
   ```
   https://indie-recommender-api.onrender.com/recommend_deep
   ```

> ⚠️ **Free tier note**: Render free services spin down after 15 minutes of inactivity  
> and take ~30 seconds to cold-start. Upgrade to the $7/mo "Starter" plan to keep it always-on.

## Adalo Custom Action Setup

In your Adalo app:

1. Go to **Settings → Custom Actions → + Add Action**
2. Fill in:
   | Field | Value |
   |---|---|
   | Name | `Get Similar Tracks` |
   | Method | `POST` |
   | URL | `https://indie-recommender-api.onrender.com/recommend_deep` |
   | Headers | `Content-Type: application/json` |
3. **Input body**:
   ```json
   {
     "track_id": "<Magic Text — Current Song > Track ID>",
     "k": 10
   }
   ```
4. **Response mapping** (Adalo "Output" tab):
   - `results[].track_id`  → Recommended Track ID  
   - `results[].title`     → Track Title  
   - `results[].artist`    → Artist Name  
   - `results[].score`     → Similarity Score

## Future Hooks (not yet implemented)

### Phase C — Hybrid: Deep Embeddings + Numeric Features

Combine CLAP embeddings with BPM/energy/mood by **vector concatenation**:

```python
# In build_embeddings.py, after computing clap_emb (512-dim):
bpm_norm   = (row["bpm"]    - BPM_MEAN)   / BPM_STD      # z-score normalise
energy_val = float(row["energy"])                          # already 0–1
numeric    = np.array([bpm_norm, energy_val], dtype=np.float32)

# Weight the contribution of numeric features vs. audio embeddings
alpha = 0.9   # 90% audio, 10% numeric
hybrid_emb = np.concatenate([alpha * clap_emb, (1 - alpha) * numeric])

# Store hybrid_emb instead of clap_emb — get_similar_tracks() needs no changes
```

Alternatively, compute two separate similarity scores and blend:
```python
audio_score   = cosine_sim(audio_emb_query,   audio_emb_catalog)
numeric_score = cosine_sim(numeric_emb_query, numeric_emb_catalog)
final_score   = 0.8 * audio_score + 0.2 * numeric_score
```

### Phase D — User Behavior Signals (Collaborative Filtering Layer)

After collecting user likes/skips from Adalo:

1. **Log events** to a simple Postgres table:
   ```sql
   user_events(user_id, track_id, event_type, created_at)
   -- event_type: 'like' | 'skip' | 'complete'
   ```

2. **Learn a user embedding** by averaging the CLAP vectors of liked tracks:
   ```python
   liked_track_ids = get_liked_tracks(user_id)
   user_emb = np.mean([embeddings[id] for id in liked_track_ids], axis=0)
   user_emb /= np.linalg.norm(user_emb)
   ```

3. **Blend** user embedding with the seed track embedding at query time:
   ```python
   blended_query = 0.5 * seed_emb + 0.5 * user_emb
   # Pass blended_query into the cosine similarity loop
   ```

4. **New endpoint** `POST /recommend_personalized`:
   ```json
   { "track_id": "TRK001", "user_id": "USR999", "k": 10 }
   ```

### Phase E — Scale to 100k+ Tracks

Replace the NumPy dot-product loop with a vector database:
- **Pinecone** (managed, easiest): drop-in replacement, same 512-dim vectors
- **Qdrant** (self-hosted, free): great for cost control
- **pgvector** (Postgres extension): if you're already using Postgres

The `get_similar_tracks()` function signature stays the same — only the internals change.
